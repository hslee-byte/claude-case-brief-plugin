# /case-brief — 판례 IRAC 요약

판례 텍스트 또는 사건번호 → IRAC 구조 요약 → 실무 시사점 → 활용 가능한 형태로 출력.

## WHEN TRIGGERED - EXECUTE IMMEDIATELY

- 파일 경로 인자 → Read로 읽고 Phase 0로
- 사건번호 인자 (예: 2024다12345) → WebSearch로 판례 검색 후 Phase 0로
- 텍스트 인자 → 바로 Phase 0로
- 인자 없으면 → AskUserQuestion:
  1. 사건번호로 검색
  2. 판례 텍스트 붙여넣기
  3. 파일 경로 입력

## Phase 0: 판례 읽기 + 맥락 파악

판례를 읽고 자동으로 파악:
- 법원 (대법원/고등법원/지방법원/헌법재판소/특허법원 등)
- 사건번호
- 선고일
- 사건 유형 (민사/형사/행정/세무/노동/특허 등)
- 당사자 구조 (원고/피고, 항소인/피항소인 등)

파악 후 AskUserQuestion:

**"[사건 유형] 판례입니다. 어떤 관점에서 정리할까요?"**
1. 원고/항소인 관점
2. 피고/피항소인 관점
3. 중립 (양쪽 논점 모두)

## Phase 1: IRAC 구조 요약

```
📋 판례 요약
━━━━━━━━━━━━━━━━
법원: [대법원 / OO고등법원 / ...]
사건번호: [2024다12345]
선고일: [2024.01.01]
사건 유형: [민사 / 형사 / 행정 / 세무 / 노동 / 특허]
결론: [원고 승 / 피고 승 / 일부 승 / 파기환송]

━━━━━━━━━━━━━━━━
■ Issue (쟁점)
━━━━━━━━━━━━━━━━
1. [핵심 쟁점 1]
2. [핵심 쟁점 2]
   (쟁점별 번호 매기기. 법원이 판단한 순서대로)

━━━━━━━━━━━━━━━━
■ Rule (적용 법리)
━━━━━━━━━━━━━━━━
쟁점 1:
- 근거 법령: [법률명 제O조 제O항]
- 기존 판례 법리: [관련 선례가 있으면 인용]
- 법원의 판단 기준: [법원이 제시한 판단 틀]

쟁점 2:
- ...

━━━━━━━━━━━━━━━━
■ Application (적용)
━━━━━━━━━━━━━━━━
쟁점 1:
- 원고 주장: [핵심 1-2줄]
- 피고 주장: [핵심 1-2줄]
- 법원 판단: [법리를 사실관계에 어떻게 적용했는지]

쟁점 2:
- ...

━━━━━━━━━━━━━━━━
■ Conclusion (결론)
━━━━━━━━━━━━━━━━
- 주문: [원고 청구 인용/기각/일부인용]
- 핵심 판시: [가장 중요한 판시 사항 1-2줄, 원문 인용]
```

요약 직후 AskUserQuestion:
1. **실무 시사점 분석** (추천)
2. 관련 판례 검색
3. 다른 형식으로 출력
4. 이 요약이면 충분

## Phase 2: 실무 시사점 (선택 시)

```
━━━━━━━━━━━━━━━━
💡 실무 시사점
━━━━━━━━━━━━━━━━

■ 이 판례가 바꾸는 것
- [기존 실무 관행과 달라지는 점]
- [앞으로 주의해야 할 점]

■ 활용 가능한 상황
- [이 판례를 인용할 수 있는 구체적 상황]
- [예: "용역 계약에서 하자보수 범위를 다툴 때"]

■ 반대 논거 (상대방이 쓸 수 있는 반론)
- [이 판례의 한계/구분 가능한 사실관계]
- [상대방이 "이 사건과 다르다"고 주장할 수 있는 포인트]

■ 관련 조문
- [적용된 법률 조문 전문 또는 핵심 부분]
```

## Phase 3: 결과물 생성

AskUserQuestion으로 출력 형식 선택:
1. **IRAC 요약** — 위 형식 그대로
2. **1페이지 브리프** — 보고용 축약본 (쟁점/결론/시사점만)
3. **서면 인용문** — 준비서면에 바로 붙일 수 있는 형태 ("... 대법원 2024. 1. 1. 선고 2024다12345 판결 참조")
4. **비교표** — 여러 판례를 입력한 경우 쟁점별 비교

파일 저장 필요 시 Write로 마크다운 파일 생성.

## 원칙

- 한국어 기본, 법률 용어는 원어 유지
- **판례 원문의 핵심 판시는 반드시 원문 인용** — 요약하되 변형하지 않음
- 쟁점 번호는 법원의 판단 순서를 따름
- 법령 인용 시 조/항/호까지 명시
- "이 판례에 의하면 ~이다"식 확정 표현 가능 (판례 요약이므로 AI 의견이 아닌 법원 판단)
- 단, 실무 시사점은 "~할 수 있다", "~가능성이 있다" 등 가능성 표현 사용
- 판례 원문, 당사자 정보는 절대 기억/저장하지 않음
- 검토 결과 상단에 항상 표시: "⚠️ AI 보조 요약입니다. 원문 확인을 권고합니다."
