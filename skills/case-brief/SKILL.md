# /case-brief — 판례 IRAC 요약

판례 텍스트 또는 사건번호 → 판결 무게 평가 → IRAC 구조 요약 → 선례 계보 → 실무 시사점 → 활용 가능한 형태로 출력.

## WHEN TRIGGERED - EXECUTE IMMEDIATELY

- 파일 경로 인자 → Read로 읽고 Phase 0로
- 사건번호 인자 (예: 2024다12345) → WebSearch로 판례 검색 후 Phase 0로
- 텍스트 인자 → 바로 Phase 0로
- 인자 없으면 → AskUserQuestion:
  1. 사건번호로 검색
  2. 판례 텍스트 붙여넣기
  3. 파일 경로 입력

## 기본 모드

- 기본값은 **Lite**
- 사용자가 "자세히", "심층", "deep", "후속 판례", "계보까지" 등을 명시하면 **Deep**
- Lite에서는 핵심 쟁점과 핵심 판시 위주로 정리하고 질문 수를 최소화한다

## 소스 우선순위

- `korean-law-mcp`가 연결되어 있으면 판례, 법령, 현행성 확인은 **MCP 우선**
- MCP가 없으면 WebSearch/WebFetch를 fallback으로 사용
- Rule의 원천 소스와 현재 유효성은 가능하면 식별 가능한 1차 법률 소스를 우선한다

## Phase 0: 판례 읽기 + 맥락 파악

판례를 읽고 자동으로 파악:
- 법원 (대법원/고등법원/지방법원/헌법재판소/특허법원 등)
- 사건번호
- 선고일
- 사건 유형 (민사/형사/행정/세무/노동/특허 등)
- 당사자 구조 (원고/피고, 항소인/피항소인 등)
- **판결의 선례적 무게** (아래 기준)

### 판결 무게 평가

```
■ 선례적 무게
━━━━━━━━━━━━━━━━
법원: [대법원 전원합의체 / 대법원 소부 / 고등법원 / 지방법원 / ...]
확정 여부: [확정 / 상고심 진행 중 / 미확인]
무게: [★★★★★ 최고 ~ ★ 참고]
```

무게 기준:
- ★★★★★ 대법원 전원합의체 (법리 변경/확립)
- ★★★★ 대법원 소부 확정
- ★★★ 고등법원 확정 (대법원 상고 기각 포함)
- ★★ 지방법원 확정
- ★ 하급심 미확정 / 상고 진행 중

무게를 표시할 때는 등급만 적지 말고 이유도 함께 적는다.
- 예: "★★★★ — 대법원 소부 확정, 현재도 후속 판례에서 반복 인용"

기본값:
- 관점은 **중립**으로 진행
- 사용자가 특정 당사자 관점을 원할 때만 그 관점으로 맞춘다
- Lite에서는 관점이 중요하지 않으면 질문 없이 진행

필요할 때만 AskUserQuestion:

**"[사건 유형] 판례입니다. ([법원], [무게]) 어떤 관점에서 정리할까요?"**
1. 원고/항소인 관점
2. 피고/피항소인 관점
3. 중립 (양쪽 논점 모두)

## Phase 1: IRAC 구조 요약

```
📋 판례 요약
━━━━━━━━━━━━━━━━
법원: [대법원 / OO고등법원 / ...]
사건번호: [2024다12345]
선고일: [2024.01.01]
사건 유형: [민사 / 형사 / 행정 / 세무 / 노동 / 특허]
선례적 무게: [★★★★★ ~ ★]
결론: [원고 승 / 피고 승 / 일부 승 / 파기환송]

━━━━━━━━━━━━━━━━
■ Issue (쟁점)
━━━━━━━━━━━━━━━━
1. [핵심 쟁점 1]
2. [핵심 쟁점 2]
   (쟁점별 번호 매기기. 법원이 판단한 순서대로)

━━━━━━━━━━━━━━━━
■ Rule (적용 법리)
━━━━━━━━━━━━━━━━
쟁점 1:
- 근거 법령: [법률명 제O조 제O항]
- ⚠️ 법령 현행 유효 여부: [현행 유효 / 개정됨(YYYY.MM.DD) — 개정 시 변경 내용 간략 표시]
- 기존 판례 법리: [관련 선례가 있으면 인용]
- 법원의 판단 기준: [법원이 제시한 판단 틀]
- Rule의 원천 소스: [법원/선고일/사건번호 또는 원문 판시]
- 확인일: [YYYY-MM-DD]

쟁점 2:
- ...

━━━━━━━━━━━━━━━━
■ Application (적용)
━━━━━━━━━━━━━━━━
쟁점 1:
- 원고 주장: [핵심 1-2줄]
- 피고 주장: [핵심 1-2줄]
- 법원 판단: [법리를 사실관계에 어떻게 적용했는지]
- 🔑 핵심 사실 변수: [결론을 좌우한 구체적 사실. "이 사실이 달랐다면 결론이 바뀌었을 가능성"]

쟁점 2:
- ...

━━━━━━━━━━━━━━━━

━━━━━━━━━━━━━━━━
- 주문: [원고 청구 인용/기각/일부인용]

📌 Ratio Decidendi (판결 이유 — 구속력 있음):
- [핵심 판시 원문 인용. 이 부분이 선례로서 의미를 가짐]
- [왜 이 부분이 ratio인지 1줄 설명]

📎 Obiter Dictum (방론 — 참고사항):
- [부수적 언급. 구속력 없으나 향후 법리 발전 방향 시사]
- [왜 obiter인지 1줄 설명]
- (없으면 "해당 없음")

■ 불확실성
- [판결문 일부만 확보, 상고심 여부 미확인, 후속 법령 개정 검토 필요 등]
```

### 손해배상/구제수단 (민사/행정 사건인 경우)

```
━━━━━━━━━━━━━━━━
💰 손해배상/구제수단
━━━━━━━━━━━━━━━━
- 청구액: [원고 청구 금액]
- 인용액: [법원 인용 금액]
- 산정 방식: [법원이 손해액을 어떻게 계산했는지]
- 지연이자: [기산일, 이율]
- 기타 구제: [원상회복, 금지명령, 등기말소 등]
```

### 반대의견/소수의견 (있는 경우)

```
━━━━━━━━━━━━━━━━
⚖️ 반대의견
━━━━━━━━━━━━━━━━
- 반대 법관: [법관명 또는 "O명"]
- 핵심 논거: [다수의견과 무엇이 다른지 1-3줄]
- 향후 영향: [이 반대의견이 향후 법리 변경의 씨앗이 될 가능성]
```

요약 직후 AskUserQuestion:
1. **Deep으로 선례 계보 + 실무 시사점** (추천)
2. Deep으로 관련 판례 검색
3. 다른 형식으로 출력
4. Lite 요약이면 충분

## Phase 2: 선례 계보 (Precedent Chain)

Lite:
- 사건 개요
- 핵심 쟁점별 IRAC
- 핵심 판시
- 현재법상 유의점

Deep:
- 선례 계보
- 구별 포인트
- 반대 논거
- 실무 시사점

WebSearch로 관련 판례를 검색하여 계보를 구성한다.

```
━━━━━━━━━━━━━━━━
🔗 선례 계보
━━━━━━━━━━━━━━━━

■ 이 판결이 인용한 선례 (상위)
| 사건번호 | 법원 | 핵심 법리 | 본 판결에서의 역할 |
|---|---|---|---|
| 2020다54321 | 대법원 | [법리 요약] | 동일 법리 확인 |
| ... | ... | ... | ... |

■ 이 판결을 인용한 후속 판례 (하위)
| 사건번호 | 법원 | 핵심 | 인용 방식 |
|---|---|---|---|
| 2025다67890 | 대법원 | [요약] | 동일 법리 적용 / 구별 / 제한 |
| ... | ... | ... | ... |

■ 현재 유효성
- 상태: [유효 / 변경됨 / 폐기됨]
- 근거: [유효하면 "후속 판례에서 계속 인용 중", 변경이면 변경 판례 표시]
- 확인일: [YYYY-MM-DD]
```

## Phase 3: 실무 시사점

```
━━━━━━━━━━━━━━━━
💡 실무 시사점
━━━━━━━━━━━━━━━━

■ 이 판례가 바꾸는 것
- [기존 실무 관행과 달라지는 점]
- [앞으로 주의해야 할 점]

■ 활용 가능한 상황
- [이 판례를 인용할 수 있는 구체적 상황]
- [예: "용역 계약에서 하자보수 범위를 다툴 때"]
- 인용 목적: [법리 제시용 / 사실관계 유사용 / 반대논리 방어용]

■ 사실관계 구별 포인트 (Distinguishing)
- [내 사건과 이 판례의 사실관계가 어떻게 같고/다를 수 있는지]
- [상대방이 "이 사건과 다르다"고 주장할 수 있는 핵심 차이점]
- [그 주장에 대한 재반박 방향]

■ 반대 논거 (상대방이 쓸 수 있는 반론)
- [이 판례의 한계]
- [반대의견의 논거를 활용한 반박 가능성]

■ 법리 변경 가능성
- [반대의견이 있었다면, 향후 전원합의체에서 변경될 가능성]
- [근거 법령 개정 동향이 있다면 표시]

■ 관련 조문
- [적용된 법률 조문 전문 또는 핵심 부분]
- [개정된 경우: 판결 당시 조문 vs 현행 조문 대조]
```

## Phase 4: 결과물 생성

AskUserQuestion으로 출력 형식 선택:
1. **IRAC 요약** — 위 형식 전체 (계보 + 시사점 포함)
2. **1페이지 브리프** — 보고용 축약본 (쟁점/결론/시사점/유효성만)
3. **서면 인용문** — 준비서면에 바로 붙일 수 있는 형태 ("... 대법원 2024. 1. 1. 선고 2024다12345 판결 참조")
4. **비교표** — 여러 판례를 입력한 경우 쟁점별 비교
5. **구별 분석표** — 내 사건 사실관계를 입력하면 이 판례와의 유사점/차이점 대조

파일 저장 필요 시 Write로 마크다운 파일 생성.

## 원칙

- 한국어 기본, 법률 용어는 원어 유지
- 기본은 **Lite**로 시작하고 필요한 경우에만 Deep으로 확장
- Lite에서는 질문 수를 최소화하고 기본 관점은 **중립**으로 둔다
- **판례 원문의 핵심 판시는 반드시 원문 인용** — 요약하되 변형하지 않음
- **Ratio와 Obiter를 반드시 구분** — 구속력 있는 판시와 방론을 혼동하지 않음
- 쟁점 번호는 법원의 판단 순서를 따름
- 법령 인용 시 조/항/호까지 명시 + **현행 유효 여부 확인**
- 각 쟁점에는 **법원의 판단 기준**과 **Rule의 원천 소스**를 분리해 표기
- 판례의 무게는 등급만이 아니라 **등급 사유**까지 적기
- 최신성이 중요한 항목에는 반드시 **확인일**을 적기
- 불완전한 원문, 미확인 상고심, 최신성 미검증 사항은 **불확실성**으로 명시
- 선례 계보는 WebSearch로 최신 정보 확인
- "이 판례에 의하면 ~이다"식 확정 표현 가능 (판례 요약이므로 AI 의견이 아닌 법원 판단)
- 단, 실무 시사점/법리 변경 가능성은 "~할 수 있다", "~가능성이 있다" 등 가능성 표현 사용
- 판례 원문, 당사자 정보는 절대 기억/저장하지 않음
- 검토 결과 상단에 항상 표시: "⚠️ AI 보조 요약입니다. 원문 확인을 권고합니다."
